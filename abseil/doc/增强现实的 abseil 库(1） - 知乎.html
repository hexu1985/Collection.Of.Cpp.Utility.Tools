缘起

abseil 是 google 开源的 C++通用库，其目标是作为标准库的补充。abseil 不但提供了标准库没有但很常用的功能，也对标准库的一些功能进行了增强设计，使用 abseil 库能使程序性能和开发效率都取得不错的提升。

abseil 库包含多个独立的子库。本篇主要是对abseil 的字符串处理子库 libabsl_strings.a 的功能进行介绍。

abseil 的编译与链接

abseil 官网上提供了使用Bazel 和 cmake 构建abseil 的方法，但在使用时，并未以常规链接库的形式使用 abseil。为保证兼容性，这里介绍以库的形式使用abseil 。首先按照文档使用 cmake 编译：

cd abseil-cpp  && mkdir build
cd build && cmake ..
make -j10

执行上述指令后，会在 build/absl 的各子目录中生成静态库，头文件也在这些文件中。

cmake 生成的 Makefile 中，并没有 install 目标，因而无法执行 make install，需要手动执行，这里假设把 abseil 的头文件要放到 /home/w/include/absl 下，库文件放到 /home/w/lib64 下：

(cd path/abseil-cpp/build/absl/  && find . -name '*.h' -print -or -name '*.inc' -print | tar --create --files-from -) | (cd /home/w/include/absl && tar xvfp -) 

cd path/abseil-cpp/build/absl/ && cp `find ./ -name libabsl_*.a` /home/w/lib64

这样就把头文件和库文件准备好了。abseil 生成的库文件众多，在使用时为了方便，可以将所有库文件都同时链接。这里以scons 构建工具为例，SConstruct 内容如下：

srcs = Split('test.cc')

target = 'a.out'
cpp_path= Split('/home/w/include')
libs = Split('pthread rt')
libs.append(Glob("/home/w/lib64/libabsl_*.a"))   # 链接abseil所有的子库
lib_path = Split('/home/w/lib64')
cc_flags = Split('-g -std=c++11')
cc = 'gcc'                                                                     
cxx = 'g++' 
link_flags = ["-Wl,--start-group"]  

Program(target, srcs, CPPPATH=cpp_path, LIBS=libs, LIBPATH=lib_path, CCFLAGS=cc_flags, CC=cc, CXX=cxx, LINKFLAGS=link_flags)

注意链接时必须加上 -Wl,--start-group 选项，这是由于abseil 的众多子库中有些子库是被其他库依赖的，需要由编译器帮忙解析依赖关系，否则会报告某些符号未定义。

当然，也可以把这些库合并成一个库来使用：

mkdir /tmp/abseil
cd /tmp/abseil
ls /home/w/lib64/libabsl_*.a | xargs -n1 ar x  # 批量解压

ar cru libabseil.a *.o   // 打包成单一的库
ranlib libabseil.a  
cp libabseil.a /home/w/lib64/

这时 SConstruct 的内容就可以简化为：

srcs = Split('test.cc')

target = 'a.out'
cpp_path= Split('/home/w/include')
libs = Split('pthread rt abseil')
lib_path = Split('/home/w/lib64')
cc_flags = Split('-g -std=c++11')
cc = 'gcc'                                                                     
cxx = 'g++' 

Program(target, srcs, CPPPATH=cpp_path, LIBS=libs, LIBPATH=lib_path, CCFLAGS=cc_flags, CC=cc, CXX=cxx)

解决了abseil 库的编译和链接问题后，我们就可以开始了解库的具体功能了。

string_view 介绍

string_view 是字符串的轻量级表示。在 C++ 中 std::string 是最常用的数据类型，在编程时如果不注意std::string 类型的临时变量生成和隐含的数据拷贝，会给性能带来很大的损失。如果我们对字符串的使用仅仅是读取，并且在使用期间字符串并不会被销毁，那么我们可以把字符串 “赋值” 给 string_view 类型， string_view 并不会进行数据拷贝，其性能要大大优于std::string 类型。

string_view 内部实际上是使用了两个字符指针来表示两个指针之间连续的字符串。需要注意的是，string_view 并不会持有数据本身，因而在使用string_view 时，必须保证底层字符串数据的生存期大于 string_view 的生存期。在很多开源代码中都能见到和string_view 类似的实现，可见其通用性。由于string_view 经常被使用，因而在 C++17 标准中已经将string_view 加入到了标准库中。abseil 为 string_view提供了十分丰富的接口，其使用方式如下：

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

int main() {
  std::string s = "Tired like a dog";
  absl::string_view sv(s);  // 用 std::string 初始化

  const char *cs = "Hello world"; 
  absl::string_view sv1(cs);  // 用字符串指针初始化
  absl::string_view sv2(cs + 6, 5); // 用字符串区间初始化，输出 "world"
  absl::string_view sv3(cs + 6, 8); // string_view 会自动判断字符串结尾，不会溢出，同样输出 "world"

  absl::string_view sv4;
  sv4 = sv3;   // string_view 之间可以直接赋值

  std::cout << sv << std::endl;
  std::cout << sv1 << std::endl; 
  std::cout << sv2 << std::endl;
  std::cout << sv3 << std::endl;
  std::cout << sv4 << std::endl;
  std::cout << sv4[2] << std::endl;  // 可以用方括号获取string_view 中的字符
  // sv4[2]  = 'i';  // ERROR：不允许通过方括号更改底层字符，因为string_view 不持有底层字符
  return 0;
}

输出：
Tired like a dog
Hello world
world
world 
world

string_view 还可以通过迭代器来读取字符串中的字符，string_view 提供了正向、反向、常量正向、常量反向等4类迭代器：

#include <iostream>
#include <string>
#include <absl/strings/str_join.h>
#include <absl/strings/string_view.h>

int main() {
  std::string s = "Tired like a dog";
  absl::string_view sv(s);

  for (auto it = sv.begin(); it != sv.end(); ++it) {  // 正向迭代器
    std::cout << *it << std::endl;
    //*it = 'h';  // ERROR: 不允许通过string_view 迭代器更改指向的数据，因为string_view 并不持有数据
  }
  std::cout << "----------" << std::endl;
  for (auto it = sv.rbegin(); it != sv.rend(); ++it) {  // 反向迭代器
    std::cout << *it << std::endl;
  }
  std::cout << "----------" << std::endl;

  const absl::string_view csv(sv);
  for (auto it = csv.cbegin(); it != csv.cend(); ++it) { // 常量正向迭代器
    std::cout << *it << std::endl;
  }

  std::cout << "----------" << std::endl;
  for (auto it = csv.crbegin(); it != csv.crend(); ++it) { // 常量反向迭代器
    std::cout << *it << std::endl;
  }

  return 0;
}

还可以获取 string_view 指向的区间的数据、长度、字符等：

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

int main() {
  absl::string_view sv1;
  if (sv1.empty()) std::cout << "sv1 is empty" << std::endl;

  const char* cs = "Tired like a dog";
  absl::string_view sv(cs + 6);  
  std::cout << sv.data() << std::endl;  // 输出 ”like a dog"，即string_view 指向的区间
  std::cout << sv.length() << std::endl; // 输出指向的区间长度
  std::cout << sv.size() << std::endl; // 同length
  std::cout << sv.front() << std::endl;  // 输出子区间首字符 'l'
  std::cout << sv.back() << std::endl;  // 输出子区间尾字符 'g'
  return 0;
}

string_view 还可以移除其前缀和后缀，只需要指定要移除的长度：

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

int main() {
  const char* cs = "Tired like a dog";
  absl::string_view sv(cs);  // 用 std::string 初始化
  sv.remove_prefix(6);  // 移除 sv 中从头开始的6个字符 "Tired "，底层字符串不变
  std::cout << sv << std::endl;  // 输出 "like a dog"

  sv.remove_suffix(4);   // 移除 sv 中从结尾开始的4个字符 " dog"，底层字符串不变
  std::cout << sv << std::endl; // 输出 "like a"
  std::cout << cs << std::endl;  // "Tired like a dog"

  sv.remove_suffix(20);  // ERROR: 超过当前 sv 长度，运行时报错
  return 0;
}

需要注意的是，这里移除前缀和后缀并不会更改string_view 指向的底层数据，这二者只是移动了string_view 内部的头部和尾部指针，并修改了 string_view 的长度，并未对底层数据作任何改动。

我们还能直接从string_view 中提取内容到字符数组中、获取 string_view 的子字符串，并且能直接交换两个string_view 类型变量的值：

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

int main() {
  const char* cs = "Tired like a dog";
  absl::string_view sv(cs);  // 用 std::string 初始化

  const char* cs1 = "hello world"; 
  absl::string_view sv1(cs1); 
  absl::string_view sv2 = sv1.substr(6, 10);  // 获取子字符串，长度超过末尾会截止到末尾处

  char carr[10] = { 0 };
  sv1.copy(carr, 4, 6);  // 从 sv1 的索引为6的位置开始复制 4个字符到 carr 中
  sv1.copy(carr, 8, 6);  // 从 sv1 的索引为6的位置开始复制 8 个字符到 carr 中，但由于超过末尾，只复制
                       // 到sv1 的末尾
  //sv1.copy(carr, 4, 25); // ERROR: 指定的位置超出了 sv 末尾，执行时报错

  sv1.swap(sv);  // 交换两个 string_view 类型的变量，底层数据不变

  return 0;
}

string_view 的比较

abseil 库中为 string_view 类型提供了丰富的比较功能。string_view 的比较和字符串比较函数 strcmp 的行为比较类似，执行字典序比较。比较结果为小于时返回 -1， 等于时返回 0， 大于时返回 1。

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

int main() {
  absl::string_view sv("hello"); 
  absl::string_view sv1("world");  

  if (sv.compare(sv1) < 0) {
    std::cout << "hello" << std::endl;
  }

  sv.compare(3, 2, sv1); // 比较从位置3开始的两个字符子串与 sv1 的大小
  sv.compare(3, 2, sv1, 1 3); // 比较 sv 和 sv1 的两个子串，子串用位置和长度表示
  sv.compare("hello"); // sv 与 const char* 型比较
  sv.compare(3, 2, "hello"); // 将sv子串与 const char* 型比较
  sv.compare(3, 2, "hello", 4); // 将sv子串与 const char* 型从头开始的子串比较, 4 是长度 

  return 0;
}

string_view 的查找

string_view 的查找功能同样十分强大，可以执行多类查找。查找函数通常返回匹配的位置，当没有匹配时，会返回 absl::string_view::npos。查找示例如下：

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

int main() {
  absl::string_view sv("hello wold"); 
  absl::string_view sv1("world");  
  auto pos = sv.find(sv1); // 从 sv 开头查找sv1 首次出现的位置
  pos = sv.find(sv1, 3); // 从 sv 的第三个位置开始查找 sv1 首次出现的位置，注意位置从 0 开始
  pos = sv.find("abc");
  pos = sv.find("abc", 3);
  if (pos == absl::string_view::npos) {  // 查找不到返回 absl::string_view::npos
    std::cout << "not found" << std::endl;
  }
  pos = sv.find('c', 3); // 从 sv 的第三个位置开始查找字符 'c' 首次出现的位置

  pos = sv.rfind(sv1, 3); // 从 sv 的第三个位置开始查找字符 sv1 最后出现的位置，返回位置为 sv1 首字符匹配的位置
  pos = sv.rfind("abc", 3); // 从 sv 的第三个位置开始查找字符串 "abc" 最后出现的位置
  pos = sv.rfind('c', 3); // 从 sv 的第三个位置开始查找字符 'c' 最后出现的位置

  absl::string_view charset("abcde");
  pos = sv.find_first_of(charset, 3); // 从 sv 的第三个位置开始查找字符集charset 中的任意一个字符首次出现的位置
  pos = sv.find_first_of("defg", 3); // 从 sv 的第三个位置开始查找字符集 "defg" 中的任意一个字符首次出现的位置
  pos = sv.find_first_of('c', 3); // 从 sv 的第三个位置开始查找字符 'c' 首次出现的位置，和 find('c', 3) 一样

  pos = sv.find_last_of(charset, 3); // 从 sv 的第三个位置开始查找字符集charset 中的任意一个字符最后出现的位置
  pos = sv.find_last_of("defg", 3); // 从 sv 的第三个位置开始查找字符集 "defg" 中的任意一个字符最后出现的位置
  pos = sv.find_last_of('c', 3); // 从 sv 的第三个位置开始查找字符 'c' 最后出现的位置，和 rfind('c', 3) 一样

  pos = sv.find_first_not_of(charset, 3); // 从 sv 的第三个位置开始查找不在字符集charset 中的任意一个字符首次出现的位置
  pos = sv.find_first_not_of("defg", 3); // 从 sv 的第三个位置开始查找不在字符集 "defg" 中的任意一个字符首次出现的位置
  pos = sv.find_first_not_of('c', 3); // 从 sv 的第三个位置开始查找不是字符 'c' 的任意字符首次出现的位置

  pos = sv.find_last_not_of(charset, 3); // 从 sv 的第三个位置开始查找不在字符集charset 中的任意一个字符最后出现的位置
  pos = sv.find_last_not_of("defg", 3); // 从 sv 的第三个位置开始查找不在字符集 "defg" 中的任意一个字符最后出现的位置
  pos = sv.find_last_not_of('c', 3); // 从 sv 的第三个位置开始查找不是字符 'c' 的任意字符首次出现的位置

  return 0;
}

string_view 传值

由于 string_view 的实现足够轻量，在作为函数形参时，string_view 类型可以直接 使用值传递，不需要使用引用传递。 string_view 类型使用值传递时，生成的代码量会更少，效率也会较高：

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

void Show(absl::string_view sv) {  // 使用值传递
  std::cout << sv << std::endl;
}

int main() {
  absl::string_view sv = "Tired"; 
  Show(sv);  // 使用值传递

  std::string str = "like";
  Show(str);  // 实参可以直接使用 std::string 类型
  Show("dog"); // 实参可以直接使用 const char* 类型

  return 0;
}

string_view 的生命期

在这里再强调一下 string_view 的生命期。由于 string_view 不持有底层数据，string_view 的有效生命期只能依赖于底层数据的生命期，当底层数据不存在时，string_view 也就失效了，此时再使用 string_view 很可能出错。例如把函数的返回值赋值给 string_view 通常不是一个好的做法，因为这种情形下很容易出现生命期问题：

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>

char* RetLocalStr() {
  char cs[] = "hello"; 
  return cs; 
}

int main() {
  absl::string_view sv = RetLocalStr();
  for (auto i = 0; i < 10; ++i) {
    std::cout << sv << std::endl;
  }

  return 0;
}

上述生命期问题并不是string_view 造成的，而是 RetLocalStr 本身的设计有问题。返回局部变量的指针或引用，通常会造成问题，基于这种易出错情况，最好不要把 string_view 直接用于函数的返回值。

字符串连接 StrCat 和 StrAppend

在处理字符串的连接时，abseil 提供了 str_cat.h 头文件。其中包含了高效处理字符串连接的功能。在进行字符串连接时，我们通常会使用 std::string 类的 ’+‘ 操作符，这一操作符会 隐含的创建 std::string 变量，这就影响了字符串连接的效率，而 '+=' 操作符不会创建临时变量，提高了字符串连接的效率：

std::string s1 = "A string";
s1 = s1 + " another string"; // 先创建一个临时变量，把 s1 复制进去，然后再把连接的部分复制进去，最后赋                           // 值给 s1
s1 += " another string";  // += 更为高效，不用创建临时变量

当一个表达式中有多个连接操作时， ‘+=' 操作符也无法避免临时变量的创建：

std::string s1 = "A string";
std::string another = " and another string";
s1 += " and some other string" + another;  // 还是需要创建临时变量

为了减少临时变量的创建从而提高效率，abseil 提供了 StrAppend 系列函数和 StrCat 系列函数来处理字符串的追加与连接。

由于 StrAppend 与 StrCat 系列函数采用了预分配策略减少了临时变量的创建，这二者的效率会比 ’+=' 操作符的效率更高。

StrCat

StrCat 可以高效的从多个组件构建一个字符串，这些组件可以是C 字符串、 std::string 类型、absl::string_view 类型、整型、浮点型、布尔型等等:

#include <iostream>
#include <string>
#include <absl/strings/string_view.h>
#include <absl/strings/str_cat.h>

int main() {
  absl::string_view sv = " hello";
  std::string str = " world";
  const char *cstr = " abc ";
  int i = 55;
  double f = 123.44678;
  std::string res = absl::StrCat(sv, str, cstr, i, f);
  std::cout << res << std::endl;

  return 0;
}

# 输出：
 hello world abc 55123.447

注意布尔值的 false 和 true 会被转换成字符串 “0” 和 “1”。 StrCat 处理浮点数时，会保留6位有效数字，注意不是6位小数，当浮点数小于等于 0.001 或大于等于 1000000.0 时，使用科学计数法表示：

#include <iostream>
#include <string>
#include <vector>
#include "absl/strings/str_cat.h"

int main() {
  std::string s1;
  double d = 288.71828333;
  s1 = absl::StrCat("hello ", d); 
  std::cout << s1 << std::endl;

  d = 3.141592653;
  std::string s2 = absl::StrCat("hello ", d); 
  std::cout << s2 << std::endl;


  d = 0.000005;
  std::cout << absl::StrCat("hello ", d) << std::endl;
  d = 1234567.0;
  std::cout << absl::StrCat("hello ", d) << std::endl;
  return 0;
}

// 输出
hello 288.718  // 6 位有效数字
hello 3.14159  // 6 位有效数字
hello 5e-06  // 科学计数法
hello 1.23457e+06 //科学计数法

整数转 16进制字符串

abseil 中的 StrCat 还支持将整数转换成16进制的字符串，并且可以设置用0或空格来填充空位（pad) 从而达到固定的宽度：

#include <iostream>
#include <string>
#include <vector>
#include "absl/strings/str_cat.h"

int main() {
  std::string s1;
  int i = 255;
  s1 = absl::StrCat("hello 0x", absl::Hex(i)); 
  std::cout << s1 << std::endl;

  s1 = absl::StrCat("hello 0x", absl::Hex(i, absl::kZeroPad4)); 
  std::cout << s1 << std::endl;

  s1 = absl::StrCat("hello 0x", absl::Hex(i, absl::kSpacePad4)); 
  std::cout << s1 << std::endl;
  return 0;
}

// 输出：
hello 0xff
hello 0x00ff   // 用 0 当作 pad，宽度总共为 4
hello 0x  ff   // 用空格当作 pad，宽度总共为 4

absl::Hex 的第二个参数是枚举值， 当用0 填充时，支持 kNoPad, kZeroPad2 到 kZeroPad16 等16种方式。当使用空格填充时，支持 kSpacePad2 到 kSpacePad16 等15种方式。用 0 和用空格填充都是把字符串扩展到对应的宽度。

不适合 StrCat 的场景

abseil 提供 StrCat 是为了加速字符串的拼接速度，但如果使用不当，StrCat 的速度优势就无法体现出来，特别是在进行字符串追加时，不应该使用 StrCat, 下面的用法就会影响 StrCat 的拼接速度，而应该使用 StrAppend：

str.append(absl::StrCat(...))  // 应该用 StrAppend 来实现对 str 的追加
str += absl::StrCat(...)
str = absl::StrCat(str, ...)  // 字符串本身的追加，应该用 StrAppend

StrAppend

StrAppend 用于在字符串尾部追加新的内容，新的内容可以是 std::string 类型、absl::string_view 类型、C 字符串、整数、浮点数、bool 值等等，在追加整数和浮点数时，其转换规则和 StrCat 一样：

#include <iostream>
#include <string>
#include <vector>
#include "absl/strings/str_cat.h"

int main() {
  std::string s1("hello ");
  absl::StrAppend(&s1, "world");
  std::cout << s1 << std::endl;

  s1 = "hello ";
  absl::string_view sv = "world"; 
  absl::StrAppend(&s1, sv); // 追加 string_view 类型
  std::cout << s1 << std::endl;

  s1 = "hello ";
  absl::StrAppend(&s1, 10000);
  std::cout << s1 << std::endl;

  s1 = "hello ";
  absl::StrAppend(&s1, absl::Hex(255, absl::kZeroPad4));  // 用0填充至 4 位宽度
  std::cout << s1 << std::endl;

  s1 = "hello ";
  absl::StrAppend(&s1, 314.1592653);  // 保留 6 位有效数字
  std::cout << s1 << std::endl;

  s1 = "hello ";
  absl::StrAppend(&s1, 0.0008);  // 小于等于 0.005 的浮点数用科学计数法
  std::cout << s1 << std::endl;

  s1 = "hello ";
  absl::StrAppend(&s1, 1000000.0);  // 大于等于 1000000 的浮点数用科学计数法
  std::cout << s1 << std::endl;

  s1 = "hello ";
  absl::StrAppend(&s1, 1000000.0, " world", " abc");  // 一次追加多项内容
  std::cout << s1 << std::endl;
  return 0;
}

// 输出
hello world
hello world
hello 10000
hello 00ff
hello 314.159
hello 0.0008
hello 1e+06
hello 1e+06 world abc

不适合 StrAppend的场景

使用 StrAppend 时，追加的字符串内容不能是字符串本身的引用、或其子串的引用，例如如下用法都是错误的：

std::string s = "foo";
StrAppend(&s, s);  // 追加字符串本身，结果未定义

std::string s = "foobar";                                                   
absl::string_view p = s;                                                    
StrAppend(&s, p);   // p 不持有底层数据，还是相当于追加字符串本身的内容，结果未定义

如果想追加字符串本身或其子串，只能用std::string 类型的 += 操作符，或先将字符串复制到另外一个std::string 类型的变量中，之后再用 StrAppend。

结语

abseil 的功能十分强大，本篇介绍了其中 strings 库 string_view 数据类型、StrCat 字符串拼接和 StrAppend 字符串追加等功能，关于字符串处理的更多功能将在下篇介绍。

使用开源库，不但要知其然，还要知其所以然。从功能和实现层面全面的了解一个库，会收获很多，因而后续将会对abseil 的实现原理进行分析。
